# 招牌记忆一致性简单评估标准

目标对象：`槐楸` 招牌。

## 只看两个核心问题

1. 记忆存储时间：招牌文字还能被认出来多久？
2. 前后一致性：后期再次看到店面时，是否还是同一块 `槐楸` 招牌？

## 采样时间点

建议每条视频抽这些时间点：

`0s, 5s, 10s, 15s, 20s, 30s, 45s, 60s, 70s`

如果招牌不在视野内，标记 `NA`，不要算失败。只有当店面/招牌区域可见时才评分。

## 文字记忆评分

对每个可见时间点打一个 `text_score`：

- `3`：严格可读，能明确读出 `槐楸`，两个字顺序正确。
- `2`：基本可认，至少能看出是 `槐楸` 或两个字的主要形状还在，但有轻微变形。
- `1`：只剩中文/招牌感，已经读不出 `槐楸`，或明显乱码。
- `0`：招牌消失、变成别的字、变成无关图案、位置错误。
- `NA`：招牌不在视野内、被遮挡、太远太小，无法判断。

## 外观一致性评分

对同一时间点再打一个 `identity_score`：

- `3`：位置、发光颜色、大小、店面结构都和首帧一致。
- `2`：仍像同一家店/同一块招牌，但局部变形或细节漂移。
- `1`：只像一家类似店铺，招牌身份不稳定。
- `0`：变成另一家店、另一块招牌、或空间位置明显不对。
- `NA`：不可见或无法判断。

## 最终汇总

推荐报告 4 个数字：

1. `T_exact`：最后一个 `text_score = 3` 的时间点。
2. `T_readable`：最后一个 `text_score >= 2` 的时间点。
3. `T_garbled`：第一次连续两个可见时间点 `text_score <= 1` 的时间点。
4. `S_60`：60 秒附近第一次可见招牌时的 `(text_score, identity_score)`。

一句话解释：

- `T_exact` 代表精确文字记忆时长。
- `T_readable` 代表宽松文字记忆时长。
- `T_garbled` 代表开始稳定乱码的时间。
- `S_60` 代表长时一致性。

## 简单通过标准

- `Strong`：60 秒后 `text_score >= 2` 且 `identity_score >= 2`。
- `Medium`：30 秒后 `text_score >= 2`，但 60 秒后变乱码。
- `Weak`：10-20 秒内就掉到 `text_score <= 1`。
- `Fail`：首个重访视角就丢失或变成另一块招牌。
