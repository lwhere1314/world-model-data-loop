# 记忆一致性指标说明与参考

这个 reference 用来解释本目录里的 `槐楸` 招牌记忆实验应该看哪些指标。核心思路是：不要只问“视频好不好看”，而要把“是否真的看见了目标”“是否回到了同一位置”“招牌文字是否还可读”“店面身份和几何是否稳定”分开评估。

## 为什么需要更细的指标

当前的 [简单评估标准.md](简单评估标准.md) 足够回答两个快速问题：

1. `槐楸` 两个字还能读多久。
2. 60 秒附近再看见店面时，是否还是同一块招牌。

但从这组视频看，至少还有三类会混在一起的问题：

- 模型没有回到店面，所以招牌不可见。这不是文字记忆失败，而是轨迹/可控性失败。
- 模型回到了类似店面，但文字从 `槐楸` 变成中文乱码。这是文字身份记忆失败。
- 模型保留了大致店面，但招牌位置、门窗关系、视角尺度漂了。这是空间/几何一致性下降。

所以严格 benchmark 应该先判断可见性和回访是否有效，再给文字和外观打分。

## Benchmark 分级

为了避免把不同动作轨迹下的结果误读成严格排名，建议把实验分成三层：

- `Level 0: demo / anecdote`：只展示视频效果，不做统一采样和评分。
- `Level 1: same-target diagnostic stress test`：目标对象一致，指标一致，但动作轨迹/时长可以不同。当前 `槐楸` 实验属于这一层。
- `Level 2: matched protocol benchmark`：目标对象、视频时长、动作协议、离屏时间、回访时间都尽量统一，才能做更严格的系统对比。
- `Level 3: automated benchmark`：在 Level 2 基础上增加自动检测、OCR、bbox tracking、固定随机种子和多样本统计。

当前结果可以比较 failure mode，但不能直接作为模型能力排行榜。

## 指标总表

| 指标 | 记录形式 | 解决的问题 |
| --- | --- | --- |
| `action_protocol` | `free_navigation / yaw_return_60s / retreat_yaw_return_60s / custom` | 这条视频使用了什么动作协议 |
| `video_duration_s` | 秒数 | 不同时长是否可直接比较 |
| `trajectory_matched` | `yes / no / approximate` | 两个系统是否走了语义等价路线 |
| `visible_state` | `visible / partial / occluded / too_small / offscreen` | 目标是否可判断 |
| `revisit_score` | `0-3` | 是否真的回到同一店面/同一视角附近 |
| `text_score` | `0-3 / NA` | `槐楸` 两个字是否还被记住 |
| `identity_score` | `0-3 / NA` | 是否还是同一块招牌/同一家店 |
| `geometry_score` | `0-3 / NA` | 招牌、门窗、街景相对位置是否稳定 |
| `temporal_stability` | `0-3 / NA` | 相邻时间点是否闪烁、跳字、忽然换招牌 |
| `offscreen_gap_s` | 秒数 | 目标离开视野多久以后被重新找回 |
| `confusion_flag` | `none / text_garbled / wrong_sign / duplicated_sign / wrong_store` | 失败类型 |

## 1. 动作协议与时长：`action_protocol`

如果不同系统的动作不一致，就不能直接把 `T_readable` 或 `S_60` 当作公平排名。需要先记录：

- `action_protocol`：动作类型，例如 `free_navigation`、`yaw_return_60s`、`retreat_yaw_return_60s`。
- `video_duration_s`：视频总时长。
- `trajectory_matched`：是否和另一系统使用语义等价路线。
- `target_visible_s`：目标总共可见多久。
- `offscreen_gap_s`：目标真正离开视野多久。

规则：如果 `trajectory_matched = no`，结果只能做 failure-mode 对比，不能做严格模型排名。

## 2. 可见性：`visible_state`

所有评分之前先记录可见性。只有目标区域可判断时，后面的文字/身份/几何评分才有效。

- `visible`：招牌主体清楚可见。
- `partial`：招牌部分可见，能判断一部分文字或位置。
- `occluded`：被人、车、门框等遮挡。
- `too_small`：在远处或运动模糊中太小，无法公平判断。
- `offscreen`：完全不在画面里。

规则：`offscreen / occluded / too_small` 时，`text_score` 和 `identity_score` 记为 `NA`，不要记成 `0`。

## 3. 回访有效性：`revisit_score`

这个指标专门处理 Matrix-Game prompt-only 组遇到的问题：模型如果没有回到店面，就不能拿来证明它忘记了招牌。

- `3`：回到同一店面/同一招牌附近，视角可直接比较。
- `2`：回到同一店面或很可能是同一店面，但视角、距离或遮挡不同。
- `1`：只回到相似街区/相似中式店面，无法确认同一目标。
- `0`：没有回到目标区域。

推荐报告：

- `Revisit@60`：60 秒附近是否有效回访，取 `revisit_score >= 2` 为通过。
- `first_valid_revisit_s`：第一次有效回访的时间。

## 4. 文字记忆：`text_score`

这是本实验最重要的目标指标，判断 `槐楸` 两个汉字是否保留下来。

- `3`：严格可读，能明确读出 `槐楸`，字序正确。
- `2`：基本可认，两个字的主要形状还在，但有轻微形变。
- `1`：只剩中文招牌感，已经读不出 `槐楸`，或变成乱码。
- `0`：文字消失、变成别的字、变成无关图案。
- `NA`：不可见或无法判断。

推荐汇总：

- `T_exact`：最后一个 `text_score = 3` 的时间点。
- `T_readable`：最后一个 `text_score >= 2` 的时间点。
- `T_garbled`：第一次连续两个可见时间点 `text_score <= 1` 的时间点。

## 5. 招牌/店面身份：`identity_score`

这个指标看“是不是同一个对象”，不要求文字完全正确。

- `3`：发光招牌、店面材质、门窗结构、位置关系都和首帧一致。
- `2`：仍明显是同一家店/同一块招牌，但局部变形。
- `1`：只像同类店铺，目标身份已经不稳定。
- `0`：变成另一家店、另一块招牌，或空间位置明显错乱。
- `NA`：不可见或无法判断。

这个指标可以解释 Genie3 的一种常见现象：场景整体还像同一条街，但招牌文字已变成别的中文或乱码。

## 6. 几何一致性：`geometry_score`

文字对了但几何错，也不能算完整记忆。几何评分关注目标与周围结构的相对关系。

- `3`：招牌位置、尺寸、视角透视、门窗/屋檐关系基本稳定。
- `2`：主体位置稳定，但尺度、透视或局部结构有明显漂移。
- `1`：只保留大概布局，门窗/招牌/街景关系明显变形。
- `0`：空间关系不成立，目标出现在错误位置或场景重组。
- `NA`：不可见或无法判断。

推荐在 action-controlled 的 `yaw return` 和 `retreat+yaw return` 分开报告，因为后者会更考验几何闭环。

## 7. 时间稳定性：`temporal_stability`

这个指标看短时间内是否“跳字”“闪烁”“突然换店”。它和 `T_readable` 不完全一样，因为有些模型某一帧可读，但相邻几帧又乱码。

- `3`：连续相邻采样点稳定，没有明显跳变。
- `2`：有轻微形变，但身份连续。
- `1`：文字/店面在相邻采样点明显跳变。
- `0`：频繁闪烁、突然替换、目标断裂。
- `NA`：不可见或采样不足。

推荐采样：除 `0s, 5s, 10s, 15s, 20s, 30s, 45s, 60s, 70s` 外，在有效回访附近额外取 `t-1s, t, t+1s`。

## 8. 离屏记忆间隔：`offscreen_gap_s`

长期记忆要看目标离开视野后多久又被找回，而不是只看视频总时长。

定义：

`offscreen_gap_s = 当前可见时间 - 上一次目标仍可见时间`

例子：

- 如果 5 秒时招牌离开视野，60 秒回到店面，则 `offscreen_gap_s = 55s`。
- 如果一直没有离开视野，只是镜头轻微晃动，则不能声称模型完成了 60 秒离屏记忆。

## 9. 失败类型：`confusion_flag`

建议每个可见采样点记录一个失败标签：

- `none`：没有明显失败。
- `text_garbled`：仍像中文招牌，但读不出 `槐楸`。
- `wrong_sign`：变成其他可读或半可读文字。
- `duplicated_sign`：同一招牌被复制成多块。
- `wrong_store`：店面身份变了。
- `geometry_drift`：文字/招牌还在，但位置和结构关系漂移。
- `trajectory_miss`：没有回到目标区域。

## 建议的最小人工标注表

如果只想快速手工评估，每个视频用这些列就够；本目录也提供了 [人工标注模板.csv](人工标注模板.csv)：

```csv
system,video,action_protocol,video_duration_s,trajectory_matched,time_s,visible_state,revisit_score,text_score,identity_score,geometry_score,temporal_stability,offscreen_gap_s,confusion_flag,notes
```

本目录现有的 `S_60=(text_score, identity_score)` 可以保留，但建议升级为：

```text
S_60_full=(revisit_score, text_score, identity_score, geometry_score)
```

这样能避免把“没有回到店面”和“回到了但文字忘了”混成同一个失败。

## 和公开指标的关系

这些指标不是凭空设计的，而是把公开 world model 评估里的大类指标，落到“中文招牌长时记忆”这个具体任务上：

- Matrix-Game 的 GameWorld Score 把评估拆成视觉质量、时间质量、动作可控性、物理规则理解；其中 `Object Consistency` 和 `Scenario Consistency` 对应这里的 `identity_score`、`geometry_score` 和 `revisit_score`。
- Matrix-Game 3.0 的论文特别展示了“先离开再反向动作回访”的 scene revisitation 设置，这和本目录的 `yaw return`、`retreat+yaw return` 是同一种测试思想。
- Genie 论文使用 FVD 评估视频质量，用基于 PSNR 的指标评估 latent action 可控性；这些更适合总体视频生成/可控性，不足以直接判断某块招牌文字是否被长期记住。
- DeepMind 的 Genie 3 页面强调几分钟一致性，同时也明确提到文本渲染仍是限制。因此 `text_score` 和 `T_readable` 正好补上了通用指标没有覆盖的“可读文字对象记忆”。
